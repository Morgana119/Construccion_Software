  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    Internet es la red global de computadoras que permite la comunicación y transferencia de datos. La World Wide Web es un servicio dentro de Internet que permite acceder a páginas web mediante navegadores y el protocolo HTTP.

    Catania, P. (2021, December 2). WWW vs Internet ¿es lo mismo? ADSLZone. https://www.adslzone.net/noticias/internet/www-vs-internet-diferencias/
  • ¿Cuáles son las partes de un URL?

    Protocolo (Hypertext Transfer Protocol Secured) -> https://

    Dominio: palabra que indentifica una o más direcciones IP -> www.ejemplo.com

    Ruta: especifica la ubicación del recurso en el servidor -> /ruta/recurso

    Parámetros: datos adicionales que identifican o filtran el recurso en el servidor -> ?query=valor

    freeCodeCamp. (2019, September 11). An introduction to HTTP: everything you need to know. FreeCodeCamp.org.

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    GET: Obtener datos del servidor. Solicitar datos de un recurso específico en el que no se modifican de ninguna manera. Las solicitudes GET no cambian el estado del recurso.

    HEAD: comprobar si el recurso está presente antes de realizar una solicitud GET. Similar a GET, pero solo devuelve los encabezados, no el cuerpo.

    POST: enviar datos a un servidor para crear un recurso.

    PUT: Reemplaza un recurso existente o lo crea si no existe.

    PATCH: Modifica parcialmente un recurso.

    DELETE: elimina un recurso.

    freeCodeCamp. (2019, September 11). An introduction to HTTP: everything you need to know. FreeCodeCamp.org.
    https://www.freecodecamp.org/news/http-and-everything-you-need-to-know-about-it/

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    No necesariamente. Un error 404 significa que el usuario intentó acceder a un recurso que no existe en el servidor, Indica que el servidor no puede encontrar el recurso solicitado. Puede ser por un error en la URL o a que la página fue eliminada.

    Sí, porque un error 500 indica un problema en el servidor, como errores en el código o fallas de configuración.

  • ¿Cuáles son las diferencias principales entre HTML4 y HTML5?

    HTML5 permite elementos semánticos como article, section.

    HTML5 elimina etiquetas obsoletas como font, center.

    HTML5 soporta APIs nativas como Canvas, WebSockets, y Geolocalización.

    HTML5 es mucho más dinámico e incluye elementos multimedia. Soporta de forma nativa el vídeo y el audio, e incluso puedes crear juegos o animaciones con él.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    caption: Da un título a tu tabla.

    col: Columna de una tabla.

Lab 3

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    !important debe ser usado de manera justificada, es importante que se tenga en cuenta que es solo para situaciones muy específicas, debido a que si se usa de manera muy frecuente dificultará el funcionamiento del código.

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    Al momento de escoger una imagen de fondo es importante tomar en cuenta varios factores, como por ejemplo:

    - La calidad de la imagen, tomando en cuenta que si es de baja resolución, se verá pixeleada, lo que le quitará calidad a la página.

    - Los colores y diseño también son importantes, más tomando en cuenta que se usará como fondo, por lo que hay que tener en cuenta cómo se verá el texto, ya que este debe ser legible.

    - Las dimensiones son relevantes para evitar que la imagen se distorcione, lo que podría causar que no se entienda y pierda calida.

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    Todo depende de a qué estemos cambiandole el formato. Para tipografía, usa rem o em. Para layouts, es mejor usar % y vh/vw. Para bordes y sombras, px es suficiente.

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    Un CSS minimizado reduce el tamaño del archivo, lo que mejora la carga de la página, haciendo que sea más rápida y obtenga mayor eficiencia.

Lab 4

  • ¿Qué diferencias y semejanzas hay entre Java y JavaScript?

    Ambos lenguajes tienen una sintaxis similar basada en C y son orientados a objetos, pero Java es un lenguaje compilado que se ejecuta en la JVM, mientras que JavaScript es interpretado en los navegadores. Java es fuertemente tipado y se usa en aplicaciones de escritorio, móviles y backend, mientras que JavaScript es débilmente tipado y se emplea principalmente en desarrollo web.

    Mozilla Developer Network (MDN). (s.f.). JavaScript Guide. https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide
  • ¿Qué métodos tiene el objeto Date?

    Algunos métodos del objeto Date en JavaScript incluyen getFullYear(), que devuelve el año en formato de cuatro dígitos; getMonth(), que retorna el mes (de 0 a 11); getDate(), que obtiene el día del mes (1-31); getHours(), que devuelve la hora (0-23); y getMinutes(), que obtiene los minutos (0-59).

    Mozilla Developer Network (MDN). (s.f.). Date. https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Date
  • ¿Qué métodos tienen los arreglos?

    Los arreglos en JavaScript cuentan con métodos como push(), que añade un elemento al final; pop(), que elimina y devuelve el último elemento; shift(), que elimina y devuelve el primer elemento; unshift(), que agrega un elemento al inicio; y map(), que crea un nuevo arreglo aplicando una función a cada elemento del original.

    Mozilla Developer Network (MDN). (s.f.). Array.prototype. https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array
  • ¿Cómo se declara una variable con alcance local dentro de una función?

    Para declarar una variable con alcance local dentro de una función, se utiliza let o const, ya que estas solo existen dentro del bloque en el que fueron definidas. Por ejemplo, let variableLocal = "Ejemplo"; dentro de una función hace que solo pueda ser utilizada ahí.

    Mozilla Developer Network (MDN). (s.f.). let. https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/let
  • ¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

    El uso de variables globales en funciones puede generar problemas como dificultades en la depuración, mayor consumo de memoria, menor reutilización del código y posibles conflictos si varias funciones modifican la misma variable, lo que puede provocar errores inesperados.

    Mozilla Developer Network (MDN). (s.f.). Global variables. https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Grammar_and_Types#variables
