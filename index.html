¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la red global de computadoras que permite la comunicación y transferencia de datos. La World Wide Web es un servicio dentro de Internet que permite acceder a páginas web mediante navegadores y el protocolo HTTP.

Catania, P. (2021, December 2). WWW vs Internet ¿es lo mismo? ADSLZone. https://www.adslzone.net/noticias/internet/www-vs-internet-diferencias/

¿Cuáles son las partes de un URL?

Protocolo (Hypertext Transfer Protocol Secured) -> https://

Dominio: palabra que indentifica una o más direcciones IP -> www.ejemplo.com

Ruta: especifica la ubicación del recurso en el servidor -> /ruta/recurso

Parámetros: datos adicionales que identifican o filtran el recurso en el servidor -> ?query=valor

freeCodeCamp. (2019, September 11). An introduction to HTTP: everything you need to know. FreeCodeCamp.org. https://www.freecodecamp.org/news/http-and-everything-you-need-to-know-about-it/

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Obtener datos del servidor. Solicitar datos de un recurso específico en el que no se modifican de ninguna manera. Las solicitudes GET no cambian el estado del recurso.

HEAD: comprobar si el recurso está presente antes de realizar una solicitud GET. Similar a GET, pero solo devuelve los encabezados, no el cuerpo.

POST: enviar datos a un servidor para crear un recurso.

PUT: Reemplaza un recurso existente o lo crea si no existe.

PATCH: Modifica parcialmente un recurso.

DELETE: elimina un recurso.

freeCodeCamp. (2019, September 11). An introduction to HTTP: everything you need to know. FreeCodeCamp.org. https://www.freecodecamp.org/news/http-and-everything-you-need-to-know-about-it/

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

POST es el método más adecuado porque envía datos de manera segura en el cuerpo de la solicitud y no en la URL, protegiendo información como la contraseña.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No ocurrió ningún error. Indica que la solicitud fue exitosa y el servidor devolvió el recurso solicitado.

HTTP response status codes - HTTP | MDN. (2024, October 18). MDN Web Docs. https://developer.mozilla.org/en-US/docs/Web/HTTP/Status#successful_responses

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No necesariamente. Un error 404 significa que el usuario intentó acceder a un recurso que no existe en el servidor, Indica que el servidor no puede encontrar el recurso solicitado. Puede ser por un error en la URL o a que la página fue eliminada.

404 Not Found - HTTP | MDN. (2024, October 26). MDN Web Docs. https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/404

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, porque un error 500 indica un problema en el servidor, como errores en el código o fallas de configuración.

500 Internal Server Error - HTTP | MDN. (2024, August 30). MDN Web Docs. https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/500

¿Cuáles son las diferencias principales entre HTML4 y HTML5?

HTML5 permite elementos semánticos como article, section.

HTML5 elimina etiquetas obsoletas como font, center.

HTML5 soporta APIs nativas como Canvas, WebSockets, y Geolocalización.

HTML5 es mucho más dinámico e incluye elementos multimedia. Soporta de forma nativa el vídeo y el audio, e incluso puedes crear juegos o animaciones con él.

¿Qué componentes de estructura y estilo tiene una tabla?

table: Son datos con más de una dimensión.

caption: Da un título a tu tabla.

col: Columna de una tabla.

colgroup: Grupo de columnas.

thead: Delimita el encabezado de la tabla.

tfoot: Delimita la parte de la tabla correspondiente al pie de página.

tbody: Delimita las otras partes del contenido de la tabla que no están en el encabezado o en el pie de página.

td: Celda con datos de una tabla.

tr: Fila de celdas en una tabla.

th: Celda de encabezado de una tabla.

The Table element - HTML: HyperText Markup Language | MDN. (2024, December 19). MDN Web Docs. https://developer.mozilla.org/en-US/docs/Web/HTML/Element/table

¿Cuáles son los principales controles de una forma HTML5?

Campos de entrada: ingresar texto en una sola línea.

Casillas de verificación: seleccionar múltiples opciones.

Botones de opción: seleccionar solo una opción dentro de un grupo.

Cuadros de selección: Menús desplegables con múltiples opciones.

Áreas de texto: Permiten ingresar texto en varias líneas.

Botones: Se utilizan para enviar, reiniciar o realizar acciones dentro del formulario.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

Planeación: Se definen los objetivos del proyecto y se asignan los recursos necesarios.

Análisis: Se recopilan los requisitos y se estudian las necesidades del sistema.

Diseño: Se crean los planes y las especificaciones detalladas para el sistema.

Implementación: Se desarrolla y programa el sistema según el diseño aprobado.

Mantenimiento: Se realizan ajustes, correcciones y mejoras al sistema en uso.

¿Cuál es el ciclo de desarrollo de sistemas de información?

Requisitos: Se recopilan las necesidades y expectativas de los usuarios para el sistema.

Diseño: Se planifica la estructura y la arquitectura del sistema.

Implementación: Se desarrolla el sistema basándose en el diseño.

Pruebas: Se verifica que el sistema funcione correctamente y cumpla con los requisitos.

Despliegue: El sistema se pone en funcionamiento y se entrega a los usuarios finales.

Mantenimiento: Se realizan actualizaciones y correcciones para asegurar el buen funcionamiento a largo plazo.

Ciclo de vida del software: todo lo que necesitas saber. (2024, June 3). Intelequia. https://intelequia.com/es/blog/post/ciclo-de-vida-del-software-todo-lo-que-necesitas-saber
