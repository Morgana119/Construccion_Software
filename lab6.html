Preguntas

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

  • Mejora la experiencia del usuario (UX): La validación en el cliente proporciona retroalimentación instantánea sin necesidad de esperar una respuesta del servidor.
  • Reduce la carga en el servidor: Al evitar solicitudes innecesarias con datos incorrectos, se optimiza el uso de recursos del backend.

Client-side form validation - Learn web development | MDN. (2024, December 31). MDN Web Docs. https://developer.mozilla.org/en-US/docs/Learn_web_development/Extensions/Forms/Form_validation

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Las validaciones en JavaScript se ejecutan en el cliente, lo que significa que un usuario puede modificar o desactivar el código JavaScript antes de enviar los datos. Por ejemplo:

  • Deshabilitar JavaScript en el navegador (usando herramientas de desarrollo como DevTools).
  • Manipular el DOM y modificar los valores directamente con document.getElementById("campo").value = "dato inválido";.
  • Interceptar y modificar la petición con herramientas como Burp Suite o la consola del navegador.
  • Enviar solicitudes maliciosas directamente al servidor usando herramientas como Postman o cURL.

WSTG - v4.1 | OWASP Foundation. (2025). Owasp.org. https://owasp.org/www-project-web-security-testing-guide/v41/4-Web_Application_Security_Testing/07-Input_Validation_Testing/05.8-Testing_for_Client_Side

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

A pesar de que JavaScript no debe ser la única capa de seguridad, sigue siendo una buena práctica porque es eficiente para una validación rápida y mejora la experiencia del usuario. Además evita errores accidentales por parte del usuario antes de enviar el formulario. No reemplaza la validación en el backend, pero sí actúa como un primer filtro.Por lo que, la seguridad real debe implementarse en el servidor, donde las reglas de validación no pueden ser modificadas por el usuario.
